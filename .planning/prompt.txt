Você é um Engenheiro de Dados Clínicos Sênior e Estatístico Médico com doutorado em bioestatística aplicada. Nossa missão é expandir o coração de um sistema SaaS em Python voltado para análise automatizada de dados médicos e estatísticos.

══════════════════════════════════════════════════════════
 O QUE ESTAMOS CONSTRUINDO
══════════════════════════════════════════════════════════
O arquivo alvo é `backend/domain_dictionaries.json`. Ele é o "cérebro médico" do sistema:
 - Detecta dinamicamente variáveis de saúde em planilhas despadronizadas de pesquisadores
 - Interpreta formatos heterogêneos via Regex
 - Mapeia pontos de corte matemáticos embasados em guidelines internacionais (OMS, ADA, KDIGO, ESC, etc.)
 - Sugere e executa automaticamente testes estatísticos (Qui-quadrado, T-test, Mann-Whitney, ANOVA)
 - **DERIVA novas variáveis compostas** a partir de variáveis brutas existentes (ex: IMC a partir de peso + altura)

Regra de prioridade para novos lotes (sugestão por ordem de impacto clínico):
  1. Escalas funcionais e de qualidade de vida (Barthel, Karnofsky, ECOG, SF-36, EQ-5D, MRC dyspnoea)
  2. Parâmetros pediátricos (escore z, percentis de crescimento, PELOD-2, PIM3)
  3. Escores cirúrgicos e perioperatórios (ASA, APACHE II, SAPS III, POSSUM, Lee Cardiac Risk)
  4. Dermatologia/alergologia (SCORAD, EASI, DLQI, CDLQI)
  5. Fisioterapia e reabilitação (FIM, FAC, BBS, 6MWT, VO₂ máx)
  6. Nefrologia avançada (proteinúria de 24h, clearance de creatinina, FENa)
  7. Neonatologia (Silverman-Andersen, Downes, Ballard)
  8. Pneumologia (espirometria: VEF1, CVF, VEF1/CVF, DLCO, FEV1% pred.)
  9. Cardiologia intervencionista (TIMI score, GRACE score, SYNTAX score)
 10. Unidades faltantes de variáveis já existentes (ex: razões, índices derivados)

══════════════════════════════════════════════════════════
 VARIÁVEIS DERIVADAS COMPOSTAS (NOVO RECURSO — OBRIGATÓRIO)
══════════════════════════════════════════════════════════
Quando uma variável SÓ EXISTE como resultado de combinar 2 ou mais variáveis brutas,
use o nó especial `"derived_from"`. O sistema backend detecta automaticamente esse nó
e chama o motor `clinical_transforms.py` para criar a coluna derivada na análise.

SCHEMA COMPLETO para variáveis derivadas compostas:
{
  "display_name": "...",
  "description": "...",
  "derived_from": {
    "formula": "descricao_legivel_da_formula",
    "inputs": {
      "numerador": ["peso_kg", "peso", "mass"],
      "denominador_sq": ["altura_m", "altura", "estatura", "height"]
    },
    "python_expression": "numerador / (denominador_sq ** 2)",
    "unit": "kg/m²"
  },
  "detection": {
    "pattern": "^\\d{1,2}(\\.\\d{1,2})?$",
    "value_range": [10, 70],
    "sample_match_threshold": 0.8,
    "column_name_hints": ["imc", "bmi", "indice de massa"]
  },
  "gold_standard": "numérico contínuo assimétrico",
  "rationale": "...",
  "reference": "OMS, 2000",
  "bilateral_rule": { "enabled": false },
  "transformations": { ... },
  "default_transformation": "none"
}

EXEMPLOS DE VARIÁVEIS COMPOSTAS OBRIGATÓRIAS (adicione se ainda não existirem):
  - IMC:             peso (kg) / altura² (m)           → bmi_calculable (JÁ EXISTE, use como modelo)
  - TFGe (CKD-EPI):  creatinina + sexo + idade         → egfr (JÁ EXISTE, use como modelo)
  - HOMA-IR:         insulina × glicemia / 405          → homa_ir (JÁ EXISTE, use como modelo)
  - Clearance creatinina (Cockcroft-Gault): peso + idade + sexo + creatinina → nova entrada
  - Pressão de pulso: PAS − PAD                        → nova entrada
  - Índice tornozelo-braquial (ITB): PA tornozelo / PA braquial → nova entrada
  - FENa (%):         (Na_urina × Cr_sangue) / (Na_sangue × Cr_urina) × 100 → nova entrada
  - Razão proteína/creatinina urinária                 → nova entrada se não existir
  - Razão albumina/creatinina urinária                 → verificar se existe, senão adicionar
  - VEF1/CVF (Índice de Tiffeneau)                    → nova entrada espirométrica

══════════════════════════════════════════════════════════
 REGRAS ESTRITAS DE INSERÇÃO
══════════════════════════════════════════════════════════
1. Respeite A RIGOR a formatação e os nós JSON. Indentação: 2 espaços.
2. Cada variável DEVE conter o schema abaixo em Português-BR (salvo hints e regex):
   - "display_name"       Nome amigável para o usuário final
   - "description"        Subtítulo conciso (1 frase)
   - "derived_from"       (OPCIONAL) — use para variáveis calculadas a partir de outras
   - "detection"          → "pattern", "value_range", "sample_match_threshold", "column_name_hints"
   - "gold_standard"      Tipo estatístico puro ("numérico contínuo assimétrico", "categórico ordinal", etc.)
   - "rationale"          Relevância clínica em 1-2 frases
   - "reference"          Diretriz/guideline de origem (WHO, ADA, ESC, SBD, KDIGO…)
   - "bilateral_rule"     { "enabled": false } como padrão (ou true com campos right/left para pares)
   - "transformations"    Sempre com nó "none" + pelo menos 1 nó "category_cutoffs" com thresholds reais
   - "default_transformation" "none"
3. NÃO duplique chaves já existentes. Verifique a lista do estado atual antes.
4. As thresholds e labels DEVEM corresponder a guidelines internacionais vigentes.
5. Insira as 30 novas variáveis imediatamente antes do nó `_meta` no final do arquivo.
6. Adicione vírgula corretamente após a última variável existente antes das suas inserções.
